# Defaults that guide, not constrain

Good defaults make the right thing easy and the wrong thing hard. Great defaults make the right thing obvious.

## What good defaults do

- Reduce decision fatigue: "Just use the default" is a valid answer
- Encode best practices: New team members inherit good patterns
- Enable escape hatches: When you need to override, it's clear how
